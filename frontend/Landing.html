System Features

Comprehensive tools for government case management

User Authentication

Secure login system with username, password, and role-based authentication (citizen, case-worker, manager).

Role-Based Access

Different roles have access to different parts of the system with appropriate permissions.

User Registration

New users can register with secure credentials and appropriate role assignments.

CRUD Operations

Create, read, update, and delete citizen records with comprehensive details including ID, name, job title, and mother's name.

Photo Upload

Supports file uploads for citizen photos, stored with the citizen's ID as the filename.

Family Links

View family relationships and linked profiles with their case history for comprehensive citizen management.

Case Management

Create, read, update, and delete cases with details like citizen ID, title, description, and status.

Case Tracking

Track case status (open, closed) with creation dates and progress history.

Appeals System

Citizens can submit complaints or appeals directly through the system.

Legal Analysis

Uses LLM to analyze laws and answer questions in a conversational chatbot interface.

Multi-Language

Detects user language (Arabic/English) and translates responses as needed.

Real-time Responses

Provides streaming, word-by-word responses for natural conversation flow.

Our Technology Stack

Powering the future of government case management

Python

High-level language used for the LLM chatbot and backend services.

JavaScript

The main language powering both frontend and backend components.

HTML & CSS

Used to create dynamic, animated frontend interfaces.

TensorFlow

Machine learning library for building deep learning models.

Node.js

JavaScript runtime for efficient server-side operations.

SQLite

Lightweight database system for case, citizen, and user data.
